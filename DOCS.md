# Ferrum | Заявка — Документация приложения

**Версия:** v2.4
**Файл:** `zajavka/index.html` (~1655 строк, single-file HTML app)
**GitHub:** `Rumsbor/ferrum-zajavka` (ветка `master`, auto-deploy на GitHub Pages)
**URL:** `https://rumsbor.github.io/ferrum-zajavka/`

## Назначение

Мобильное веб-приложение для управления закупками печного оборудования Ferrum. Визуализирует данные из Google Таблицы "Анализ продаж", позволяет редактировать заказы, управлять складами и пересчитывать заявку.

## Архитектура

- **Single-file HTML** — весь CSS, HTML и JavaScript в одном файле `index.html`
- **Тёмная тема** — CSS-переменные: `--bg: #0f1117`, `--card: #1a1d27`, и т.д.
- **Mobile-first** — адаптивный дизайн, `user-scalable=no`, touch-оптимизация
- **Бэкенд:** Google Apps Script Web App (POST/GET)
- **Fallback:** Google Visualization API (GViz) для чтения данных напрямую из таблицы
- **Без фреймворков** — чистый JavaScript, никаких зависимостей

## Ключевые функции JavaScript

### Загрузка данных
- `loadData()` — основная загрузка: Web App → local `data.json` → GViz API
- `loadFromGViz()` — fallback через Google Visualization API

### Рендеринг
- `renderAll()` → `applyFilters()` + `updateSummary()`
- `renderList()` — генерация HTML списка товаров
- `renderSettings()` — заполнение полей настроек
- `renderBreakdown(i)` — таблица складов для товара
- `updateSummary()` — обновление сводки + объёма в настройках

### Фильтрация
- `applyFilters()` — поиск + фильтр → `filteredData` → `renderList()`
- `setFilter(el)` — переключение фильтра

### Взаимодействие
- `toggleItem(i, event)` — раскрытие/свёртка карточки
- `toggleBreakdown(i)` — раскладка по складам
- `toggleWhDetail(itemIdx, whName, trEl)` — детали склада (пики + мин. остаток)
- `sortBreakdownCol(i, col)` — сортировка таблицы складов

### Редактирование
- `saveManual(i)` — сохранение ручного значения заказа (POST `setManual`)
- `clearManual(i)` — очистка ручного значения
- `saveMinStock(itemIdx, whName)` — сохранение мин. остатка склада (POST `setMinStock`)

### Настройки
- `toggleSettings()` — открытие/закрытие панели
- `applyParams()` — POST `setParams` (дни + скидка)
- `saveWarehouses()` — POST `setWarehouses`
- `toggleWarehouse(el)` — переключение активности склада
- `updateDiscount()` — live-обновление скидки в сводке
- `togglePriceView()` — переключение РРЦ/скидка

### Даты
- `updateDateHints()` — вычисление и отображение дат по дням назад/вперёд (и в настройках, и в шапке)
- `formatDate(d)` — форматирование в `DD.MM.YYYY`
- `parseDateDMY(s)` — парсинг `DD.MM.YYYY` → `Date`
- `parseServerDate(val)` — универсальный парсинг даты от сервера

### Пересчёт заявки
- `recalculate()` — главная функция пересчёта:
  - Без целевого объёма: `applyParams` → `saveWarehouses` → POST `recalculate` → `loadData`
  - С целевым объёмом: **метод секущих** (см. ниже)
- `doServerRecalc(btn, days)` — одна итерация серверного пересчёта, возвращает объём
- `recalcPeaks()` — пересчёт пиков продаж (POST `recalcPeaks`)

## Алгоритм подбора целевого объёма (метод секущих)

Задача: подобрать `daysBack` чтобы объём заявки ≈ целевому.

**Формула заказа** (в таблице): `CEILING(MAX(0, мин_ост + продажи - остаток), кратность)`
- Продажи ∝ дни → связь прямая: больше дней = больше продаж = больше заказ

**Алгоритм:**
1. Точка 1 известна: `(d1=currentDaysBack, v1=currentVol)`
2. Оценка d2: `d2 = round(d1 × target / v1)` (прямая пропорция)
3. Серверный пересчёт → получаем реальный `v2`
4. Если `|v2 - target| / target ≤ 5%` → готово
5. Иначе метод секущих: `d3 = d1 + (target - v1) × (d2 - d1) / (v2 - v1)`
6. Серверный пересчёт → финальный результат

**Максимум 2 серверных вызова.** Поле целевого объёма очищается после пересчёта.

## UX-решения

- **Бейдж заказа**: синий градиент для товаров с заказом, серый для нулевого. Показывает пачки (N×M) если packSize > 1
- **Индикаторы**: ▼ красный = дефицит (stock < minStock), ▲ зелёный = в норме
- **Live-даты**: при изменении дней назад/вперёд даты обновляются мгновенно (без пересчёта)
- **Мин. остаток**: поле `readonly`, активируется по клику (не вызывает клавиатуру при открытии панели)
- **Цена**: клик на сводку или карточку переключает РРЦ ↔ скидка
- **Пики**: показываются в панели склада, сворачиваются после 4 периодов
- **Toast**: зелёный (success) или красный (error), 2.5 сек

```bash

```

**Ограничение:** CORS не позволяет тестировать API Apps Script с localhost. Полное тестирование — только на GitHub Pages.
